API quickstart

One endpoint, two languages. Set ANTHROPIC_API_KEY in your environment; the SDK reads it automatically. Default to claude-sonnet-4-6.

python
import anthropic

client = anthropic.Anthropic()   # reads ANTHROPIC_API_KEY

msg = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system="You are a helpful assistant.",
    messages=[{"role": "user", "content": "Hello, Claude"}],
)
print(msg.content[0].text)
javascript / typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();   // reads ANTHROPIC_API_KEY

const msg = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  system: "You are a helpful assistant.",
  messages: [{ role: "user", content: "Hello, Claude" }],
});
console.log(msg.content[0].text);

Production checklist

Before you point real users at an endpoint, tick these off:

Keys server-side

In env / secret manager. Never in the client or the image.

Retries & timeouts

SDK retries 429/5xx; set sane request timeouts.

Input validation

Length caps, block secrets, refuse out-of-scope.

Evals

A golden set that runs on every change.

Logging

Trace id, tokens, latency — redact sensitive data.

Prompt caching

Cache large stable context to cut cost.

Rate limits

Throttle per-user; handle 429 gracefully.

Cost monitoring

Track token spend; alert on anomalies.

Developer glossary

API key
Your secret credential for calling the model. Lives on the backend, in an environment variable or secret manager — never in client code.
Token
The chunk of text a model reads and writes in — roughly ¾ of a word. Billing is per token, split into input (what you send) and output (what it writes).
System prompt
Top-level instructions that set the model's role and rules for the whole conversation, separate from the user's messages.
Structured output
Constraining the response to a shape your code can parse — usually JSON matching a schema — so you don't have to scrape prose.
Tool use
Giving the model functions it can call. It returns a tool_use request; your code runs the function and sends the result back.
Embedding
A vector of numbers that captures the meaning of a piece of text, so similar meanings land close together in space.
Vector DB
A database built to store embeddings and find the nearest ones fast — e.g. pgvector, Pinecone, FAISS.
RAG
Retrieval-Augmented Generation: fetch relevant chunks from your data and add them to the prompt so the model answers from your documents.
Chunking
Splitting documents into small passages before embedding, so retrieval returns focused, relevant pieces.
Eval
An automated test for model output — a golden set of inputs and expected qualities, run on every change to catch regressions.
Guardrail
A check around the model call — input validation, output filtering, scope limits — that keeps unsafe or off-topic requests out.
Prompt caching
Marking a large, unchanging chunk of context as cacheable so repeat calls reuse it far more cheaply than resending it.
Rate limit
The cap on requests or tokens per minute. Exceeding it returns a 429; back off and retry (the SDK does this for you).
