🏛️ Level 4 Capstone

Ship a RAG-Powered Assistant

Prove you can take AI from a code snippet to a running service. You'll build a small document-Q&A API — ingest a few docs, embed and store them, retrieve, answer with Claude — then deploy it to a public URL and back it with an eval set and a cost note.

The goal

A live endpoint — say POST /ask — that answers questions from documents you supply, with the API key server-side, a small eval set proving it works, and a short note on what it costs to run. Every Level 4 skill, wired into one deployable service.

Build steps

① Scaffold the backend

Start from the one-endpoint FastAPI server. Confirm it answers a plain question before adding retrieval.

starter · server.py
import anthropic
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()
client = anthropic.Anthropic()   # ANTHROPIC_API_KEY from the environment

class Ask(BaseModel):
    question: str

@app.post("/ask")
def ask(body: Ask):
    answer = rag_answer(body.question)   # you'll write this in step ④
    return {"answer": answer}

② Ingest & chunk your docs

Pick 3–10 documents (an FAQ, a policy, a README). Split them into small passages — one idea per chunk retrieves better than one giant blob.

starter · chunk
def load_chunks(path: str) -> list[str]:
    text = open(path, encoding="utf-8").read()
    # simplest chunking: split on blank lines / paragraphs
    return [p.strip() for p in text.split("\n\n") if p.strip()]

chunks = load_chunks("faq.txt")
print(len(chunks), "chunks")

③ Embed & store

Turn each chunk into a vector and keep the vectors alongside their text. Start in-memory; swap in a real vector DB later.

starter · embed + store
import numpy as np
from sentence_transformers import SentenceTransformer

embed = SentenceTransformer("all-MiniLM-L6-v2")
vectors = embed.encode(chunks)     # in prod: pgvector / Pinecone / FAISS

④ Retrieve top-k & answer with Claude

Find the most similar chunks and paste them into a grounded prompt. Instruct Claude to answer only from the context.

starter · rag_answer
def rag_answer(question: str) -> str:
    q = embed.encode([question])[0]
    sims = vectors @ q / (np.linalg.norm(vectors, axis=1) * np.linalg.norm(q))
    top = [chunks[i] for i in sims.argsort()[-3:][::-1]]   # top-3
    context = "\n".join(f"- {c}" for c in top)

    msg = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=512,
        system="Answer ONLY from the context. Cite the lines you used. "
               "If the answer isn't in the context, say you don't know.",
        messages=[{"role": "user",
            "content": f"Context:\n{context}\n\nQuestion: {question}"}],
    )
    return msg.content[0].text

⑤ Add an eval set

Write 5–10 golden cases from your own docs. Run them on every change so you notice regressions before your users do.

starter · evals.py
GOLDEN = [
    {"q": "How long do refunds take?", "must_include": "5 business days"},
    {"q": "What is the return window?", "must_include": "30 days"},
]

passed = sum(
    case["must_include"].lower() in rag_answer(case["q"]).lower()
    for case in GOLDEN
)
print(f"{passed}/{len(GOLDEN)} passed")

⑥ Containerize & deploy

Wrap it in the Dockerfile from Lesson 4.4 and ship to a public URL. Keep the key in a secret store, never in the image.

starter · deploy
gcloud run deploy rag-assistant \
  --source . \
  --region asia-south1 \
  --set-secrets ANTHROPIC_API_KEY=anthropic-key:latest \
  --allow-unauthenticated
# → note the https://...run.app URL it prints; that's your deliverable

⑦ Write a cost note

Log usage.input_tokens and usage.output_tokens on a handful of requests and describe, qualitatively, what a typical answer costs and which lever (caching, smaller model, tighter max_tokens) you'd pull first if traffic grew. No need for exact figures — reference the pricing page.

What to hand in

Three things:

  • A repo link — the backend, Dockerfile, and eval script.
  • A live URL — your deployed /ask endpoint, answering from your docs.
  • Eval results — the pass rate of your golden set, plus the one-paragraph cost note.
◆ Success criteria — you've passed when…

✓ Answers cite the retrieved context and say "I don't know" when it's absent.
✓ The API key is server-side — nowhere in the client or the container image.
✓ There's an eval set that runs and reports a pass rate.
✓ It's deployed to a public URL that responds to a real request.

▸ Stretch goal

Add one of: tool use (let the assistant call a real function — order lookup, availability check — mid-answer), or streaming (stream the response token-by-token to the client with client.messages.stream(...) so replies feel instant).
