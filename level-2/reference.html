Advanced prompt patterns

Five levers that turn a good answer into a reliable one. Stack them one at a time and stop when the output is dependable:

Personas / roles

"You are a careful contracts lawyer." Primes vocabulary, caution, and priorities.

Few-shot examples

Show 2–3 input→output pairs. The model copies your pattern better than any description.

Chain-of-thought

"Think step by step before answering." Trades a little speed for a lot of accuracy on reasoning.

Structured output

"Return valid JSON / a markdown table with columns X, Y, Z." Machine- and eye-readable.

Delimiters

Fence pasted data in """ or <tags> so it's never mistaken for an instruction.

Plan-then-execute

"Give me a plan first; don't start until I say go." Catches wrong assumptions early.

Plus the reusability habits: save your context once in a Project/system prompt, keep a tiny eval set, and run automations in draft mode until they earn your trust.

Reusable prompt library

System prompt skeleton

"ROLE / AUDIENCE / FORMAT / VOICE / RULES (ask one question if a detail is missing; never invent facts)."

Structured extract

"Extract [fields] from the text below into a table. Quote sources exactly; write 'not specified' if a field is missing — never guess."

JSON, no prose

"Return ONLY valid JSON matching this schema: {…}. No commentary before or after."

Plan first

"Give me a numbered plan and what you'll need from me. Don't start until I reply 'go'."

Review this code

"Explain this to a non-programmer, then list what could break (bad input, edge cases) and how to guard each. Don't rewrite yet."

Compare two docs

"List only substantive changes between v1 and v2 (rights, timelines, money). Show old → new and why each matters."

Tiny eval run

"Run my prompt on these 6 test inputs. Per case: PASS/FAIL + why. Flag anything invented, omitted, or softened."

Automation spec

"TRIGGER → AI ACTION → DRAFT only (never auto-send) → human approves. Write it as steps I can build in Zapier/Make."

Weekly update

"Turn these notes into Shipped / In progress / Risks. Max 5 bullets each, verb-first, no filler, no invented metrics: [notes]"

Triage & label

"Label each item as [categories], matching these examples: [few-shot pairs]. Then label the new batch."

Plain-English glossary

System prompt
A standing instruction block that shapes every reply in a Project, custom GPT, or Gem. Write it once, reuse it forever.
Few-shot
Giving the model a few worked input→output examples so it copies your pattern. (Zero-shot = no examples.)
Chain-of-thought
Asking the model to reason step by step before answering. Improves accuracy on multi-step or trade-off problems.
Structured output
Requesting the answer in a fixed shape — JSON, a table, a schema — so it's reliable to read and reuse.
Agent
An assistant that plans, acts, checks its result, and loops until a goal is met. In Level 2 you're the loop; Level 5 closes it.
Automation
A saved "when X happens, do Y" chain (Zapier/Make) or a scheduled prompt that runs without you pressing send.
Token budget
The limited amount of text a model can read and write in one go. Long contexts and outputs cost more of it — you'll meet tokens properly in Level 3.
