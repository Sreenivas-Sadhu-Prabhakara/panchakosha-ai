🏛️ Level 2 Capstone

Automate a Real Workflow

Prove you can turn a chat habit into a work system. Pick one task you actually do every week, build it a reusable assistant, wire a single automation, and measure the time you get back.

The goal

Choose one recurring work task and, over a week, turn it into a reusable setup: a Project (or custom GPT / Gem) with a saved system prompt, a small prompt library of the variations you use, and one automation that removes a manual step. Then measure the time saved. You'll exercise every Level 2 skill — advanced prompting, reusable context, and connecting tools — on something real.

The four steps

① Pick the task & baseline it

Choose something recurring, context-heavy, and shape-stable (weekly report, standard replies, meeting notes, lead follow-ups). Before automating anything, time yourself doing it the old way — that's your baseline.

starter
Here are 5 recurring tasks in my work week: [list]. For each, rate fit for a
reusable AI setup (1-5) and estimate minutes it takes me now. Recommend the
ONE with the best time-saved-to-effort ratio, and say why.

② Build the reusable assistant

Create a Project (or custom GPT / Gem) and give it a real system prompt — role, audience, format, voice, and rules for missing information. Add a reference file if the task needs one.

starter · system prompt
ROLE: You are my assistant for [the task].
AUDIENCE: [who reads the output].
FORMAT: [exact shape — headers, table, length].
VOICE: [tone]. Never inflate; be honest about what's unfinished.
RULES:
- Turn my raw notes into the format above.
- If a required detail is missing, ask ONE question, then wait.
- Never invent facts, names, or numbers I didn't provide.

③ Assemble a small prompt library

Save 3–5 reusable prompts for the variations you actually hit — the short version, the escalation, the "make it formal", the follow-up. Keep them in the Project or a note so future-you just picks one.

starter
Given my system prompt above, draft 4 reusable follow-up prompts I'll likely
need for this task (e.g. shorten, make more formal, add a data table, write a
chaser). Keep each to one line I can paste. Label each with when to use it.

④ Wire one automation & measure

Add a single automation that removes a manual step — a Zapier/Make zap or a scheduled prompt — running in draft mode so you approve before anything sends. Then do the task the new way and record the new time.

starter · automation spec
TRIGGER: [when it should run — new row / new email / Monday 8am].
ACTION (AI): [what to generate, using my system prompt].
OUTPUT: Create a DRAFT only — never auto-send.
GUARDRAIL: A human reviews and approves before it goes out.
Write this as clear steps I can build in Zapier or Make.

What to hand in

One page (Google Doc, Notion, or a note) titled "[Task] — automated" containing: the task and its before/after time, your system prompt, your prompt library, a description (or screenshot) of the automation, and two lines on what broke or surprised you.

◆ Success criteria — you've passed when…

✓ A reusable assistant exists with a real system prompt you can run again next week.
✓ Your prompt library has at least 3 saved, labelled variations.
✓ One automation runs and does something useful — safely, in draft/approve mode.
✓ You can state a measured time saved (before vs. after), not a guess.
✓ No sensitive data went anywhere you'd regret in a breach.

▸ Stretch goal

Give your assistant a tiny eval set — 5 tricky real inputs and what a good output must include — and run them to prove the setup holds up. Then hand the whole thing to a colleague and see if they can use it without you explaining. A system a teammate can run is the real finish line.
