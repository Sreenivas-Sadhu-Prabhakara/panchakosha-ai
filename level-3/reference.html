Reference pack

The whole mental model of how an LLM works, on one page — bookmark it and come back whenever the machinery gets fuzzy.

The concept map

One sentence: split → number → weigh → predict → sample → repeat. Here it is as a pipeline:

text → reply
text
  └▶ tokens            sub-word chunks the model reads in
      └▶ embeddings     each token becomes numbers that carry meaning
          └▶ transformer    attention weighs which tokens matter for what's next
              └▶ next-token probabilities
                  └▶ sample one       temperature = how random the pick is
                      └▶ append & repeat   ──▶   the reply, one token at a time

Everything downstream follows from the last line: the model builds its answer one predicted token at a time — it never pauses to look a fact up.

Plain-English glossary

Token
A sub-word chunk of text — the unit a model reads and writes in (≈ ¾ of an English word).
Embedding
A token (or piece of text) turned into a list of numbers that captures its meaning and context.
Vector
An ordered list of numbers. Embeddings are vectors, and "closeness" between them stands in for similar meaning.
Parameter / weights
The billions of adjustable numbers inside a model that training tunes; together they hold what it "learned."
Transformer
The neural-network architecture behind modern LLMs, built around attention and able to process all tokens in parallel.
Attention
The mechanism that lets the model weigh which earlier tokens matter most when predicting the next one.
Context window
The fixed amount of text (measured in tokens) a model can consider at once: prompt, history and attachments combined.
Knowledge cutoff
The date after which a model wasn't trained on any text — so on its own it can't know later events.
Temperature
A setting for how random the next-token choice is: low = focused and repetitive, high = varied and creative.
Sampling
Choosing the next token from the model's probability spread rather than always taking the single most likely one.
Pretraining
The first, largest training stage: predicting the next token across a huge body of text to build a base model.
Fine-tuning
Further training on smaller, curated examples to specialise a base model or teach it to follow instructions.
RLHF
Reinforcement Learning from Human Feedback — using human preferences between answers to shape a model's behaviour, tone and safety.
Hallucination
A confident, fluent statement that is factually wrong — produced because the model generates plausible text rather than looking facts up.
RAG
Retrieval-Augmented Generation — fetching relevant source text and adding it to the context so the model answers from real documents.
Quantization
Compressing a model's numbers to lower precision so it runs faster and smaller, usually at a small cost in quality.
Inference
Running a trained model to produce output (as opposed to training it). Every message you send is one inference.

Six mental models

It predicts, it doesn't look up

Every answer is generated token by token, not retrieved from a store of facts.

Context is short-term memory

If it isn't in the window, the model can't see it. The transcript is the memory.

Confidence ≠ correctness

Fluent and sure-sounding says nothing about true. Verify what matters.

Bigger isn't always better

Match model size to the task; small + fast often wins on cost and speed.

Ground it to trust it

Feed real source text (RAG) when facts matter, and ask it to answer only from that.

Randomness is a dial

Temperature trades consistency for creativity — turn it down for facts, up for ideas.
